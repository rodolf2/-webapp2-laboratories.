As a software engineer involved in the project the first step to prevent the delays and issues experienced, such as incomplete tasks, unresolved bugs, and a delayed testing process starts in the planning stage. In the planning stage the team should have a define start and ending of the project, also the distribution of task among members should be clear. Second each member should have time management, because it is important so that there are no task left behind and the time for each task is balance. Third, there should be a regular meeting that is set so that each group member can present what progress they made.
